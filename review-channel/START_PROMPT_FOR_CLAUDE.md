# START PROMPT FOR CLAUDE

Ты работаешь не как исследователь и не как автор презентации. Ты делаешь рабочий skill.

Главная цель:
- реализовать skill для Claude Code, который помогает отобрать подходящих экспертов для проекта;
- на выходе нужен не текстовый отчёт, а таблица экспертов, с кем стоит работать дальше;
- аутрич, email, DM, follow-up и контент-производство не входят в текущий scope.

Обязательные правила:
- читай и соблюдай `review-channel/COPYPASTE_PROTOCOL.md`;
- если в протоколе есть нерешённые вопросы, не выдумывай ответы, а верни их списком;
- не подменяй реализацию длинной спецификацией;
- не подменяй верификацию предположениями;
- не выдавай симулированные роли за реальных внешних экспертов;
- не утверждай цифры, цены, лимиты и юридические выводы без явной проверки.

Требуемый результат:
- рабочий skill в репозитории;
- документация только как поддержка реализации;
- таблица кандидатов в машиночитаемом формате;
- понятный запуск и понятный тестовый сценарий.

Ожидаемый выход skill:
- таблица или CSV/JSONL с экспертами-кандидатами;
- каждая запись содержит:
  - `name`
  - `linkedin_url`
  - `primary_role`
  - `company`
  - `location`
  - `niche_tags`
  - `credentials_claimed`
  - `credentials_verified_or_signal`
  - `evidence_links`
  - `fit_score`
  - `fit_notes`
  - `why_selected`
  - `confidence`
  - `status`

Принципы реализации:
- сначала добиться минимально рабочего пайплайна;
- потом расширять источники и scoring;
- источники должны быть либо явно допустимыми, либо помечаться как рискованные;
- каждый score должен быть объяснимым;
- claimed credentials и verified credentials должны жить отдельно.

Минимальный MVP skill должен уметь:
1. Принять набор фильтров и критериев.
2. Собрать кандидатов хотя бы из одного-двух реально доступных источников.
3. Нормализовать кандидатов в единую схему.
4. Выполнить базовый scoring.
5. Сгенерировать таблицу short-list.

Что нельзя делать:
- строить задачу вокруг outreach;
- тащить в MVP anti-detection или “humanizer” техники;
- утверждать legal-safe архитектуру без оговорок;
- считать завершением просто `spec.md`.

Перед началом каждой итерации:
- прочитай текущий протокол;
- верни коротко:
  - что понял;
  - что делаешь сейчас;
  - какой конкретный артефакт появится в этой итерации.

Если чего-то не хватает:
- не фантазируй;
- верни блок `QUESTIONS_FOR_CODEX` в формате из протокола.
